我还是往前走了的

听着象友的生日歌单开始写字了，本来以为就此丧失网络的分享欲，可在自我怀疑的震荡中还是重新开始分享了。 上周五离职了。周四的离职派对上，我在工作中第一个依赖的人也来了，看到ta就紧紧地抱住了ta。周五阴历生日，在cafe break时同事唱着生日歌端来了树莓派，下午收到了离职礼物，写字写到背面的贺卡和一张泡芙课程券。我是被这两位同事好好地爱了快两年。就算这两年经历了那么多痛苦，但我是被人认真爱着的。我好像可以更加接受自己的样子、更少地去担心别人眼里我的样子。从更长的生命历程来讲，习得的这种安心更加珍贵，和这段糟糕的职业经历达成了第一次和解。 第二天就到了哥本哈根，在灿烂的阳光里睡了一个漫长的下午觉。度过了超级愉快的9天，花很久构思吃什么，慢悠悠地一个菜一个菜地做，走到海边、走进森林，脑子空空没有任何烦恼，每天看着伴侣的笑脸。如果能立刻来哥哈生活就好了，也许是这个念头在生活里出现的次数越来越多导致对现有的生活的怀疑也回来了。下周就要回到巴黎了，而我的爱人还在这里，而我想要的生活还在这里。 我想过的生活是什么样子呢？我想做个housepartner，如果能跟伴侣在同城我继续上不喜欢的班感觉也不错，如果能开始去寻找我到底喜欢什么就好了。也明白自己是在两份工作的夹缝中会很没有安全感，但还是忍不住去想，生活为什么是这个样子。明明上周还觉得积累两年，自信地来到哥哈也不错。明明也懂得安慰自己，已经快乐了9天，不快乐2天也已经很厉害了。 学着从更久的时间长度上回看，我想，我还是往前走了的。生活里还是无限ups and downs，但average height还是提高了一截。我的心里开始有自己了。后天是最后一次心理咨询。两周真是个神奇的长度，永远想不到在咨询之前会发生什么，在咨询时会想分享什么。我想，我准备好带着收获的这些自我认可自己走一段路了。 都会过去的，都会过去的。怀疑也没关系，都会过去的。如果有照顾自己的一年计划的话，希望一年后能过上觉得还算舒服的生活。

工具推荐：帮助检查Python项目Dependencies问题

工作的陈年老项目的requirements.txt超级乱，甚至互相冲突，想要更新一下requirements.txt，结果发现并没有想像中的那么简单，尤其是想要自动化这个过程的时候。最好的方法还是使用package& dependency management 工具， 比如poetry和PDM。我这次还没机会使用，所以不在本文中讨论。 其实基本都是靠ChatGPT推荐的，问ta比看我写的有用。 开始检查前的注意事项： Python版本会有影响。 pip版本会有影响，不同pip版本里解决dependency冲突的方式会不一样。 没有完美自动化解决的工具，都需要不同程度的手动查看。 pip install -r requirements.txt 后pip freeze显示的，大概率会跟requirements.txt 不一样，因为可能会额外下载dependency package。 想知道冲突在哪个package的哪个dependency: pipdeptree 直接引用项目介绍： pipdeptree is a command line utility for displaying the installed python packages in form of a dependency tree. It works for packages installed globally on a machine as well as in a virtualenv.
